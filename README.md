# FX Position Maturity Dashboard

FX Forward / SWAP 포지션의 만기금액(KRW) 분포를 시각화하는 GitHub Pages 대시보드입니다.

## 📺 대시보드 URL

```
https://{username}.github.io/fx-histogram/
```

---

## 📁 Repository 구성

```
fx-histogram/
└── index.html        ← 대시보드 본체 (이 파일만 있으면 됩니다)
```

엑셀 파일은 repo에 올리지 않고, 대시보드에서 직접 업로드합니다.

---

## 🚀 GitHub Pages 설정

1. Repository → **Settings** → **Pages**
2. Source: **Deploy from a branch**
3. Branch: **main** / folder: **/ (root)**
4. Save → 약 1분 후 URL 활성화

---

## 📋 엑셀 파일 규격

각 계정별 파일을 따로 준비합니다. 시트명은 무관하며 **첫 번째 시트**를 자동으로 읽습니다.

| 컬럼명 | 내용 | 비고 |
|--------|------|------|
| `기준일자` | 데이터 기준일 | **첫 번째 데이터 행**에만 입력, 나머지 행은 공백 |
| `자산구분명` | FX Forward / SWAP 구분 | 참고용, 필터 없음 |
| `통화키_지급` | Sell 통화 | USD, EUR, JPY 등 |
| `만기수취금액` | KRW 수취금액 | 숫자 또는 콤마 포함 문자 모두 인식 |
| `만기일자` | 만기일 | YYYY-MM-DD 또는 YYYY.MM.DD |
| `거래상대방명` | 거래상대방 이름 | 없으면 거래상대방 섹션 미표시 |

> 기준일자 이전 만기 포지션은 자동 제외됩니다.  
> `기준일자` 컬럼이 없거나 비어있으면 오늘 날짜를 기준으로 사용합니다.

## 📊 사용 방법

### 1. 계정 탭 선택 후 파일 업로드
- **일반계정** / **특별계정** 탭을 각각 전환하여 파일 업로드
- 두 계정 파일을 독립적으로 업로드·유지 (탭 전환 시 데이터 유지)
- 업로드 시 파일의 `기준일자` 컬럼을 읽어 Info Bar에 자동 표시
- ✕ 버튼으로 개별 초기화

### 2. 거래상대방 섹션
- 파일에 `거래상대방명` 컬럼이 있으면 히스토그램 아래 자동 표시
- 도넛 차트 + 순위 테이블 (거래상대방 · 만기금액 · 비중%)
- 컬럼이 없으면 해당 섹션은 비어있는 상태로 유지

### 3. X축 구분 선택

| 옵션 | 예시 |
|------|------|
| 월별 | 2025.06, 2025.07 … |
| 분기별 | 2025 Q3, 2025 Q4 … |
| 반기별 | 2025 H2, 2026 H1 … |
| 연별 | 2025, 2026 … |

### 3. 통화 필터
- **전체**: 모든 통화 합산 표시
- 개별 통화 선택: 복수 선택 가능, 차트·테이블 즉시 연동

## 🔄 데이터 업데이트 방법

`index.html`은 수정할 필요 없습니다. 새 데이터가 생기면 대시보드에서 파일을 재업로드하면 됩니다.
